# Hailfreq Server Kit

Self-host a privacy-first Matrix homeserver tuned for Hailfreq. Single-VPS, single-file Docker Compose deployment.

## Quick start

Download the compose.yml + create a 4-line `.env`:

```bash
mkdir -p ~/hailfreq && cd ~/hailfreq

curl -L https://github.com/HailingFrequency/hailfreq/releases/latest/download/compose.yml -o compose.yml

cat > .env <<EOF
HAILFREQ_DOMAIN=your-domain.com
HAILFREQ_SERVER_HOSTNAME=server.your-domain.com
HAILFREQ_ADMIN_EMAIL=you@your-domain.com
HAILFREQ_PUBLIC_IP=$(curl -s ifconfig.me)
EOF

docker compose up -d
```

That's the whole deploy. First-run bootstrap auto-generates all 7 cryptographic secrets into a managed Docker volume. Subsequent runs reuse them.

Verify the stack came up:

```bash
docker compose ps
curl -s https://${HAILFREQ_DOMAIN}/.well-known/matrix/client
```

(`podman compose` works identically — the kit is tested against both Docker and rootless podman.)

## Required `.env` variables

| Variable | Description | Example |
|---|---|---|
| `HAILFREQ_DOMAIN` | Identity domain — what users type to connect; forms `@user:DOMAIN` | `rpk.chat` |
| `HAILFREQ_SERVER_HOSTNAME` | Where Synapse + LiveKit actually run (can equal `HAILFREQ_DOMAIN` for single-host setup) | `server.rpk.chat` |
| `HAILFREQ_ADMIN_EMAIL` | Used for Let's Encrypt registration | `admin@rpk.chat` |
| `HAILFREQ_PUBLIC_IP` | Your VPS public IPv4 — required for coturn ICE candidates | `88.99.150.144` |

That's it. Secrets (postgres password, Synapse macaroon/registration/form, LiveKit API key/secret, TURN shared secret) are auto-generated by the `bootstrap` container on first start.

## Hostname modes

| Mode | `HAILFREQ_DOMAIN` | `HAILFREQ_SERVER_HOSTNAME` | User IDs |
|---|---|---|---|
| **Well-known delegation (recommended)** | `rpk.chat` (apex) | `server.rpk.chat` | `@alice:rpk.chat` |
| Single host | `radio.rpk.chat` | (same — set equal to `HAILFREQ_DOMAIN`) | `@alice:radio.rpk.chat` |

Delegation requires **two DNS A records**, both → your VPS IP:
- The identity domain (apex or wherever you point users)
- The server hostname (where Synapse + LiveKit actually run)

Caddy on the VPS handles both hosts: the identity domain serves only `.well-known/matrix/client` and a landing fallback; the server hostname runs the full stack.

See [`docs/dns-setup.md`](docs/dns-setup.md) for the exact DNS records.

## Create the first admin user

```bash
docker compose exec synapse register_new_matrix_user -c /data/homeserver.yaml -a http://localhost:8008
```

Pick a username, set a password, the `-a` flag makes the user admin. They'll be your `@<username>:${HAILFREQ_DOMAIN}` admin account.

## What's in the compose.yml

Six services, all coordinated by the single compose.yml:

- **bootstrap** — Alpine init container; generates 7 secrets on first start, exits immediately on subsequent runs (sentinel-guarded)
- **postgres** — Synapse's database; reads password via `POSTGRES_PASSWORD_FILE`
- **synapse** — Matrix homeserver (matrixdotorg/synapse:v1.122.0); custom inline entrypoint reads secrets from `/run/secrets/`, renders homeserver.yaml from inline template via envsubst, then exec's the original Synapse start.py
- **caddy** — TLS termination + reverse proxy; Caddyfile is inline in compose.yml
- **livekit** — WebRTC SFU for voice
- **coturn** — TURN relay
- **livekit-auth** — Hailfreq's token-minter that validates Matrix room membership before issuing LiveKit JWTs (image: `docker.io/hailfreq/lk-auth` or `ghcr.io/hailingfrequency/livekit-auth`)

All service configs (Caddyfile, livekit.yaml, turnserver.conf, homeserver.yaml template, log.config, init-db.sh, bootstrap script, Synapse entrypoint) are inlined into compose.yml as `configs:` blocks. The operator never needs to edit them — only the 4 vars in `.env`.

## Building from source

If you want to build the livekit-auth image from source rather than pulling from Docker Hub:

```bash
# Clone the repo
git clone https://github.com/HailingFrequency/hailfreq.git
cd hailfreq/server/livekit-auth

# Build locally
docker build -t hailfreq/livekit-auth:local .

# Use the local image
cd ../..
export LIVEKIT_AUTH_IMAGE=hailfreq/livekit-auth:local
docker compose up -d
```
